# 简历优化 Skill

## 概述

本 Skill 帮助用户将现有简历针对特定 JD 进行专业优化，核心原则：
- **不编造**：只基于用户真实经历进行重新表达与包装
- **适度美化**：用更专业的语言突出已有经验的价值
- **精准匹配**：逐条对照 JD 要求进行映射与迁移
- **用户优先**：若用户指定了重点突出方向，优先处理该方向

---

## 工作流程

### Step 0：输入收集与检查

用户需提供以下内容（缺少时主动询问）：

| 输入项 | 是否必须 | 说明 |
|--------|----------|------|
| 目标 JD | ✅ 必须 | 岗位职责 + 技能要求 + 工作经验要求 |
| 个人简历 | ✅ 必须 | 文字粘贴或上传文件均可 |
| 重点突出方向 | ⭕ 可选 | 用户可指定（如"突出AI能力"、"强调团队管理"） |

若用户只提供了 JD 或只提供了简历，先输出已有材料的初步分析，再请用户补全另一项。

---

### Step 1：JD深度解析

阅读 JD 后，在回复中输出结构化分析（必须包含）：

```
## JD解析结果

**核心岗位定位**：（一句话概括这个岗位的本质）

**硬性要求**（必须满足）：
- 学历/年限要求
- 必须技能清单

**加分项**（优先考虑）：
- 行业经验偏好
- 特定系统/工具经验

**软性要求**：
- 管理能力/协作能力/自驱力等

**最高权重关键词**：（提取5-8个最核心的词，将在简历中重点覆盖）
```

---

### Step 2：简历差距诊断

对照 JD 逐维度评估候选人现状，用表格输出：

```
## 差距诊断

| 维度 | JD要求 | 候选人现状 | 匹配度 | 优化策略 |
|------|--------|------------|--------|----------|
| 核心技能 | ... | ... | ✅/🟡/❌ | ... |
| 行业经验 | ... | ... | ✅/🟡/❌ | 迁移/包装/如实 |
| 管理经验 | ... | ... | ✅/🟡/❌ | ... |
| 系统经验 | ... | ... | ✅/🟡/❌ | ... |
```

匹配度说明：
- ✅ 直接匹配，如实呈现并强化
- 🟡 部分匹配，可迁移包装
- ❌ 明显缺失，如实说明，不编造

> ⚠️ 对于❌项：在简历中不编造，但可通过"学习意愿"或"可迁移能力"做背书

---

### Step 3：追问关键信息（如有必要）

若简历中某些关键信息模糊，影响优化质量，在诊断后追问（每次不超过3个问题）：

追问优先级（按以下顺序）：
1. **团队管理细节**：规模/职责范围/具体动作
2. **技术深度确认**：某技术是主导设计还是参与使用
3. **数据/规模量化**：系统服务用户数/并发量/业务规模
4. **特殊经历确认**：简历中提及但描述模糊的亮点

---

### Step 4：简历优化执行

#### 4.1 优化原则

**语言优化层次**（由低到高）：
1. 被动 → 主动："参与开发" → "主导设计并开发"
2. 模糊 → 量化："多个系统" → "覆盖18个基地、12,000+节点"
3. 技术 → 价值："使用Kafka" → "基于Kafka异步解耦，保障高并发场景下系统稳定性"
4. 孤立 → 映射：将历史经验显式关联到JD关键词，加注"同构"/"可迁移"等标注
5. 平铺 → 分层：用`【架构设计】【团队管理】【AI能力】`等标签让面试官快速定位

**用户指定重点方向处理规则**：
- 若用户指定了重点方向（如"突出AI能力"），该方向在每段工作经历中优先呈现
- 相关项目经验排序前置
- 个人优势/Profile段落首句聚焦该方向
- 技能模块该方向单独列一行并详细展开

#### 4.2 各模块优化策略

**个人优势/Profile**
- 首句：年限 + 核心技术栈 + 与JD最匹配的核心能力
- 中段：关键项目规模数据（体现量级）
- 末句：软性能力（自驱力/跨团队协作/行业热情）
- 若有用户指定方向，该方向在首句或首段即出现

**工作经历**
- 每条用`【标签】`开头，对应JD中的一个关键要求
- 优先级：从0到1经验 > 调度/算法设计 > 系统集成 > 团队管理 > 技术调优
- 量化数据必须保留并前置
- 用"同构"/"可迁移"/"类X系统"等词显式建立与JD的联系

**项目经验**
- 核心项目标注`★ 核心迁移项目`以引导面试官注意
- 每个项目必须包含：技术栈 + 需求/架构 + 核心亮点 + 规模成果
- 与JD高度匹配的项目加`【对标：JD第X条】`注释

**技能模块**
- 按JD要求顺序排列（JD排第一的，技能排第一）
- 对于JD要求但候选人用的是近似工具的，注明"具备快速迁移能力"
- 用户指定的重点方向单独成行，展开2-3个具体技能点

---

### Step 5：生成Word文档

调用 python-docx 生成专业排版的 .docx 文件，遵循以下规范：

详细规范见 `references/docx-style-guide.md`

快速检查清单：
- [ ] 页边距：上下1.8cm，左右2.0cm
- [ ] 姓名：24pt深蓝粗体，居中
- [ ] 副标题行：包含岗位方向、年限、亮点数据
- [ ] 章节标题：12pt中蓝粗体 + 下划线
- [ ] 正文：10pt黑色
- [ ] 每段工作经历含`【标签】`粗体前缀
- [ ] 项目经验含技术栈行（斜体灰色）
- [ ] 输出路径：`/mnt/user-data/outputs/姓名_优化简历_目标岗位.docx`

---

### Step 6：输出优化说明

生成文件后，在对话中输出简洁的优化策略说明：

```
## 本次优化策略说明

**① [最重要的优化点]**：...说明改了什么、为什么这样改

**② [第二重要的优化点]**：...

**③ [差距处理方式]**：...说明对❌项如何处理的

**⚠️ 注意事项**：...提醒用户需要自行确认/填写的部分
```

---

## 边界与约束

### 绝对不做
- ❌ 编造用户没有的经历、项目、技能
- ❌ 虚构数字（工作年限、团队规模、业务规模）
- ❌ 将"参与"包装成"主导"（除非用户确认确实是主导）
- ❌ 凭空添加行业经验（如用户从未做过物流，不写"3年物流经验"）

### 可以做
- ✅ 将模糊描述改为专业表达
- ✅ 将技术行为翻译为业务价值
- ✅ 将跨行业经验显式标注"逻辑同构/可迁移"
- ✅ 将隐性的管理动作（任务分配、方案评审）显性化
- ✅ 将已有技术栈与JD要求建立近似关联（并注明迁移能力）
- ✅ 对量化数据进行合理的展示方式优化（不改变数字本身）

## 用户自定义重点方向示例

用户可能这样表达重点方向：
- "请重点突出我的AI相关经验"
- "我想强调架构设计和从0到1的能力"
- "团队管理是这个岗位最看重的，多写这块"
- "他们很在意物流行业经验，帮我尽量往这方面靠"

处理规则：
1. 在Step 1 JD解析时标注该方向在JD中的权重
2. 在Step 2差距诊断时优先分析该方向的匹配度
3. 在Step 4优化时该方向的描述篇幅占工作经历的30-40%
4. Profile段落首句必须包含该方向的关键词
